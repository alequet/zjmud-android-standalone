# 武学扩展计划

本文档固定 Android 单机版 zjmud 的武学扩展范围、候选优先级、聚合规则和实施门槛。
它记录 2026-07-21 对当前武学库与 `/Users/pan/Downloads/skill` 下载库的审查结论，作为后续
设计和实现基线；下载库只是候选来源，不是可直接导入的可信上游。

状态：**规划完成，尚未实施。** 当前 runtime 中没有加入本文候选武学。

## 1. 目标与边界

本计划的目标不是扩大技能数量，而是补足当前 zjmud 缺少的终局成长路线：

- 以多门既有武学演练、参悟形成的聚合武学；
- 当前体系缺少的顶级刀法、佛门剑掌和跨门类武学；
- 有明确成长、代价、反噬或路线选择的高阶防御武学；
- 能融入现有门派、任务、研读和武学修养体系的完整能力。

以下内容不在范围内：

- 玩家自创武学、巫师测试技能和只供展示的管理技能；
- 仅有夸张名称、单个固定高伤招式或明显私服数值的模板；
- 与当前 zjmud 同 ID 或同定位冲突、却没有足够差异化的旧实现；
- 为移植一门武学而引入整套无关门派、转生或私服属性系统；
- 直接修改 `app/src/main/assets/runtime/zjmud-runtime.zip`。

APK 实际内容以生成的 runtime 为准；武学实现必须作为 UTF-8 可审计源文件或补丁保存在
`tools/mudlib/`，再由 `tools/import_zjmud.sh` 转码、校验并确定性重建 runtime。

### 2.2 冲突处理原则

1. 当前 zjmud 已存在同 ID 武学时，以当前实现为准。
2. ID 不同但名称、故事来源和战斗定位相同的，也必须进行概念冲突审查。
3. 下载库中的九阳神功、乾坤大挪移、葵花体系、龙象体系等不能覆盖当前版本。
4. 可以吸收旧文件中的单个机制或文本设计，但不得借此暗中替换当前技能行为。
5. 当前不存在并不代表适合新增；仍需通过完整度、依赖、数值和获取路线审查。

## 3. 候选评审标准

每门候选武学按以下维度评审：

| 维度 | 通过条件 |
| --- | --- |
| 差异化 | 能补充当前没有的战斗定位，而不是换名复制 |
| 完整度 | 主技能、绝招、内功功能、状态和解除逻辑形成闭环 |
| 依赖兼容 | 所需基本技能、特殊技能、condition、角色字段和 daemon 在当前 zjmud 中存在 |
| 数值兼容 | 使用当前命中、伤害、忙乱、内力和防御量级，不依赖私服超额属性 |
| 获取闭环 | 明确通过演练、参悟、任务、秘籍或门派获得，不能仅靠管理员注入 |
| 可恢复性 | 临时属性、callout、condition 和失败分支能正确回滚，不永久污染角色 |
| 可测试性 | 能建立静态依赖检查、LPC 编译检查和确定性战斗 smoke |

候选分级含义：

- **A：优先实施。** 体系价值明确，当前库能够承接，修订范围可控。
- **B：保留设计。** 值得做，但下载源码必须明显重写或需要额外世界内容。
- **C：不进入计划。** 与当前重复、来源不合适、代码质量差或收益不足。

## 4. 聚合武学机制

### 4.1 当前协议

当前 `yanlian` 要求目标技能实现：

- `query_sub_skills()`：返回前置技能及等级；
- `get_ready()`：开始前校验；
- `get_finish()`：完成时校验和叙事；
- 演练完成后取前置技能平均等级作为新技能等级；
- 删除 `query_sub_skills()` 中的全部前置技能。

该协议适合“数门同类武学熔为一门”的替代式聚合，例如六脉神剑或刀剑归真。它不适合
只要求玩家理解多门武学、但最终技能不能替代全部前置用途的能力。

### 4.2 必须增加的第二种模式

扩展前应将聚合分为两种显式模式：

| 模式 | 行为 | 适用案例 |
| --- | --- | --- |
| 消耗式演练 | 校验并删除全部指定前置技能，新技能接管其用途 | 刀剑归真、天道拳/剑/刀、武神剑 |
| 保留式参悟 | 校验前置技能但不删除，按规则授予或解锁新技能 | 七弦无形剑、中华傲诀、阴阳十二重天 |

建议保持现有技能默认使用消耗模式，以免改变 `dragon-strike` 和 `six-finger` 行为；新增技能
通过显式接口声明保留模式或声明需要删除的前置子集。无论采用何种接口，必须满足：

- 不能删除基本内功、基本招架等通用基本技能；
- 不能删除最终技能不支持的必要用途，例如七弦无形剑不能删除蛤蟆功后让玩家失去内功；
- 开始前一次性校验所有源码和依赖，完成前再次校验角色状态；
- 只有授予新技能成功后才能删除旧技能；
- 失败、打断、断线或进程终止不得留下半完成状态；
- 参悟来源和等级计算必须确定，不依赖 mapping 的无序遍历结果。

## 7. 第三梯队：大型重构项目

### 7.1 阴阳十二重天 `yinyang-shiertian`

设定上由九阳、九阴、北冥、凌波、乾坤、降龙、六脉等顶级武学融会而成；这些概念前置
当前均有承接基础，因此它适合作为长期最高层参悟目标。

原源码不能直接使用：

- 没有实现 `query_sub_skills()`，并非可工作的聚合技能；
- 依赖当前不存在的 `4zhuan`、`5zhuan`、`zhuanfinal` 等私服转生状态；
- 多用途、双击、完全防御等能力叠加过多，容易压倒全部现有武学；
- 若删除七门顶级前置，会摧毁玩家既有门派和绝招生态。

计划定位：保留式参悟的长期终局目标。先设计十二层逐层解锁、能力预算和互斥关系，再重写
技能；不得从下载文件直接开始修补。

设定上由靛沧海、金晨曦、血穹苍三卷合一，适合做分卷收集和多阶段参悟。

- `valid_learn()` 和 `practice_skill()` 永久失败，没有实际学习路线；
- 没有聚合协议；
- 难度 9000、双击和动作数值来自不同量级；
- 三卷的独立技能与最终能力没有完成状态转换。

计划定位：以三卷任务和保留式参悟重做，不直接复制主文件。每卷先成为可独立使用、可测试
的内功分支，最终宝鉴只增加新的组合能力，避免简单叠加三份数值。

### 7.3 明玉功 `mingyu-gong`

下载库包含 11 个子模块，覆盖内功、护体、群攻、拳脚和剑法，是相对完整的大型技能包；但

计划定位：只有在决定新增移花宫门派、NPC、秘籍和任务线后实施。它不是可孤立加入武学库

## 8. 防御武学专项决策

### 8.1 魔神真身 `moshen-zhenshen`

结论：**不作为独立通用武学原样移植。**

原因：

- 学习门槛、`valid_damage()`、完全抵伤和招架曲线与当前 `jingang-buhuaiti` 高度相似；
- “无法无天”主要是在护甲强化之外增加 `defense / 3`，不足以支撑新技能身份；
- `practice_skill()` 开头直接返回，后续练习代码永远不可达；
- 对少林攻击者直接失效，缺少合理机制和叙事解释；
- 与天魔护体、天魔金身共同加入会形成多门同质完全免伤技能。

可吸收内容：将“无法无天”的攻防强化设计用于邪派防御路线，或者将魔神真身作为金刚不坏
的互斥邪派叙事分支。不能让玩家同时叠加金刚不坏、魔神真身和天魔护体。

### 8.2 天魔金身 `tianmo-jinshen`

天魔金身的十三层成长和走火入魔比魔神真身更有独立价值，但原文件仍需重写：

- `nuoyi.c` 使用未初始化的 `level`；
- `matrial-cognize` 拼写错误；
- `difficult_level()` 被拼成 `diffcult_level()`；
- 被动完全抵伤没有合理内力成本；
- “金身战气”实际复制乾坤大挪移式牵引，文本和技能定位不一致。

最终防御线只保留一个新增位置：以天魔金身十三层为成长骨架，吸收“无法无天”的短时
攻防强化，重新实现成功率、内力消耗、冷却和反噬。金刚不坏体继续以当前 zjmud 为准。

## 10. 数值与设计约束

所有新增武学遵守以下长期约束：

1. 不允许无消耗、无冷却、无对抗判定的完全免伤或完全闪避。
2. `double_attack` 不能再与高概率四连击、无条件附伤同时叠加；多段攻击必须共享总伤害预算。
3. 忙乱时长设置硬上限，不能按未经封顶的伤害或技能等级直接换算。
4. 临时 `apply/*` 属性必须保存实际增加值并精确回滚，不能在结束时重新读取已变化的技能等级。
5. condition 必须在当前 runtime 中存在、可加载、可解除；不得静默引用下载库私有状态。
6. 攻防判定优先复用当前 zjmud 的 `COMBAT_D` 和同层级顶级武学公式。
7. 门槛应使用当前已有的经验、武学修养、基本技能、门派贡献和任务状态。
8. 不引入 `reborn/times`、私服转数、未注册特殊属性作为必要条件。
9. 聚合武学的强度来自用途整合和新战术，不等于把全部前置技能数值相加。
10. 新武学必须保留现有顶级武学的选择价值，不能成为所有角色的唯一最优解。

## 11. 实施顺序

### P0：扩展聚合协议

- 定义消耗式演练与保留式参悟接口，并保持当前两个聚合技能兼容。
- 增加前置依赖静态校验、确定性等级计算、失败回滚和管理员诊断。
- 为聚合过程增加断线、打断、技能缺失和完成失败测试。

### P1：最小纵向切片

1. 移植刀剑归真，验证消耗式演练、获取、映射、绝招和保存恢复。
