# zjmud 剧情设计工作区

本目录用于沉淀现有世界设定、剧情资产盘点和后续剧情方案。它与 `docs/` 的工程文档分开：
`docs/` 说明 Android 移植和运行架构，本目录只讨论世界、人物、叙事与任务设计。

当前前十章采用 V2“无面局”悬疑版本。`16-前十章重构总纲-无面局.md` 是跨篇章反派、身份反转
和解谜结构的权威总纲；各章末尾的 V2 小节覆盖同章早期样板中的平淡冲突表述。

## 基线与证据

- 盘点日期：2026-07-15。
- 前十章 V2 重构日期：2026-07-17。
- 主要分析对象：`app/src/main/assets/runtime/zjmud-runtime.zip`，即当前 APK 实际发布的 mudlib。
- 对照资料：仓库 `README.md`、`docs/ARCHITECTURE.md`、`tools/import_zjmud.sh` 及
  `tools/mudlib/` 下的单机版补丁。
- runtime 是 GB18030/GBK 时代的 LPC 代码。统计时按 `.c` 文件、继承声明和目录结构归类；
  数量用于判断内容重心，不代表每个文件都能从正常游玩路径进入。
- 本文把“代码存在”“守护进程已登记”“玩家当前可触发”分开描述。仅存在于目录中的内容，
  不自动视为已启用内容。

## 使用约定

1. 新剧情先提交设计稿，再改 LPC；设计稿使用 `03-剧情设计原则与模板.md` 的模板。
2. 已上线事实写入“现状”，推断和建议必须明确标注，不能混成既有设定。
3. 新增跨地域或改变著名人物命运的内容时，先更新世界观基线和冲突清单。
4. 每次 runtime 基线变化后，复核 `story`、`event`、`quest`、`fuben`、`d/`、`kungfu/class/`
   六类入口，并更新盘点日期。
5. 所有跨章分支必须在设计稿中声明收束点。普通章节遵守 70% 至 80% 共同骨干、每章最多一个
   且非必要不产生跨章核心选择，并禁止新增永久独立主线；没有收束方案的分支不得实现。
